### US-1.6 摄像头 + 语音录制

| 项目 | 内容 |
|------|------|
| **故事编号** | US-1.6 |
| **用户故事** | 作为一名**教育工作者**，我需要在录制屏幕操作的同时录制我的摄像头画面和语音讲解，并将它们合成到最终视频中（画中画效果），以便学生能够看到我的面部表情和听到我的讲解，增强教学效果。 |
| **关键交互** | 启用摄像头录制选项、选择摄像头设备、启用音频录制、实时预览摄像头画面、调整摄像头位置和大小（左下角/右下角/左上角/右上角）、调整音频音量 |
| **技术实现** | `getUserMedia()` 获取摄像头和麦克风流 + `AudioEncoder` 编码音频 + OPFS 存储 `camera.bin` 和 `audio.bin` + Composite Worker 画中画合成 + 音画同步（基于 timestamp） |
| **是否完成** | ⏳ 未开始 |

**详细说明**：
- **摄像头录制**: 使用 `getUserMedia({ video: true })` 获取摄像头流，使用 `VideoEncoder` 编码为 VP9/H.264
- **音频录制**: 使用 `getUserMedia({ audio: true })` 获取麦克风流，使用 `AudioEncoder` 编码为 Opus（192kbps）
- **OPFS 存储**:
  - `camera.bin` - 摄像头视频数据
  - `camera-index.jsonl` - 摄像头视频索引
  - `audio.bin` - 音频数据
  - `audio-index.jsonl` - 音频索引
- **画中画合成**: 在 Composite Worker 中将摄像头画面叠加到屏幕录制画面上
- **音画同步**: 基于 `timestamp` 字段对齐视频和音频轨道
- **存储开销**: 音频约 1.4 MB/分钟（192kbps），摄像头视频约 5-10 MB/分钟（720p 30fps）

**依赖关系**：
- ✅ F-1 (真人出镜) - 本功能是 F-1 的完整实现
- ✅ F-2 (画中画合成) - 需要画中画合成能力

**预计工作量**: 7-11 天

| 项目 | 内容 |
|------|------|
| **故事编号** | US-1.8 |
| **用户故事** | 作为一名**产品演示者**，我需要录制鼠标移动轨迹，并在编辑时能够切换不同的鼠标指针样式（默认箭头/手型/放大镜/自定义图片），以便制作更专业的产品演示视频，帮助观众跟随我的操作步骤。 |
| **关键交互** | 自动录制鼠标轨迹（无需用户操作）、编辑时选择指针样式、调整指针大小、预览指针效果、隐藏/显示鼠标轨迹 |
| **技术实现** | CaptureController API (`oncapturedmousechange` 事件) + OPFS 存储 `mouse.jsonl` + Composite Worker 绘制鼠标指针 + 指针样式库（SVG/PNG） |
| **是否完成** | ⏳ 未开始 |

**详细说明**：
- **鼠标事件捕获**: 使用 `CaptureController` API（Chrome 109+）
  ```typescript
  const captureController = new CaptureController()
  captureController.oncapturedmousechange = (event) => {
    const mouseEvent = {
      timestamp: performance.now() * 1000,  // 微秒
      x: event.surfaceX,
      y: event.surfaceY,
      isInside: event.surfaceX !== -1 && event.surfaceY !== -1
    }
    // 存储到 OPFS
  }
  ```
- **OPFS 存储**: `mouse.jsonl`，每行一个鼠标事件
  ```jsonl
  {"timestamp":0,"x":100,"y":200,"isInside":true}
  {"timestamp":16666,"x":105,"y":205,"isInside":true}
  ```
- **指针样式**:
  - **默认箭头**: 黑色箭头指针
  - **手型**: 手指点击图标
  - **放大镜**: 放大镜图标
  - **自定义**: 用户上传的 PNG/SVG 图片
- **合成渲染**: 在 Composite Worker 中根据时间戳绘制鼠标指针
- **存储开销**: 约 50 字节/事件，60fps × 300秒 = 18,000 事件 = 900 KB
- **浏览器支持**: Chrome 109+, Edge 109+

**依赖关系**：
- ✅ 已验证可行性 (`lab/CaptureController/`)
- ✅ 基于现有 OPFS 存储架构
