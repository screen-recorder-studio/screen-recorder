> 文档版本：v1.0 | 日期：2026-02-10
> 适用项目：Screen Recorder Studio Chrome Extension (v0.6.7)

## 一、战略背景

在 AI 技术快速普及的背景下，Chrome 录屏扩展可以通过引入 AI 能力实现 **从工具到智能平台** 的升级。AI 不仅能提升产品体验、降低用户流失，还能为付费转化提供强有力的价值锚点。

### 1.1 为什么要引入 AI？

1. **差异化竞争** — 市面主流录屏工具（Screenity、Awesome Screenshot）尚未深度集成 AI
2. **付费转化抓手** — AI 功能天然适合作为 Premium 功能收费
3. **用户价值升级** — 从 "录制视频" 升级为 "智能生产内容"
4. **降低使用门槛** — AI 辅助让非专业用户也能产出高质量内容

### 1.2 商业化前提

| 条件 | 当前状态 | 优先级 |
|------|---------|--------|
| 用户基数 | 需要增长 | P0 先解决留存 |
| 付费系统 | 未搭建 | P1 同步推进 |
| 后端服务 | 无（纯本地）| P1 按需搭建 |
| API 成本控制 | 未规划 | P1 设计用量模型 |

## 三、P0 — 快速上线功能（1-3 周）

### 3.1 ⭐ AI 自动字幕生成

**价值**：字幕是视频内容的刚需，特别是教程、演示类录屏。

**技术方案**：
```
录制音频 → Whisper API / 本地 Whisper → 文本 → SRT/VTT 字幕 → 导出叠加
```

**实施路径**：
1. 录制时同步提取音频轨道（当前已捕获 `MediaStream`）
2. 录制完成后，音频发送到 AI 服务端进行语音识别
3. 返回带时间戳的文本，生成 SRT 字幕文件
4. Studio 中支持字幕预览和编辑
5. 导出时将字幕烧录到视频或附带 SRT 文件

**技术选型**：

| 方案 | 优点 | 缺点 | 成本 |
|------|------|------|------|
| OpenAI Whisper API | 高精度，支持 99 种语言 | 需要网络，有隐私考量 | $0.006/分钟 |
| 本地 Whisper (WASM) | 完全本地，隐私友好 | 速度较慢，模型较大 | 免费 |
| Web Speech API | 浏览器原生，免费 | 精度一般，仅实时 | 免费 |
| 混合方案（推荐）| 本地优先，高精度可选云端 | 实现复杂度稍高 | 可控 |

**推荐方案**：混合架构
```typescript
// ai-subtitle-service.ts
class AISubtitleService {
  async generateSubtitles(
    audioBlob: Blob,
    options: SubtitleOptions
  ): Promise<SubtitleTrack> {
    if (options.mode === 'local') {
      // 使用 Whisper WASM 本地推理
      return await this.localWhisper(audioBlob);
    } else {
      // 云端高精度转录（Premium 功能）
      return await this.cloudWhisper(audioBlob, options.language);
    }
  }
}
```

**商业模式**：
- 🆓 免费：每月 30 分钟本地转录
- 💰 Premium：无限云端高精度转录 + 多语言支持

**投入**：2-3 周（含后端 API）
**预计转化率**：5-10% 免费用户转付费

### 3.2 ⭐ AI 内容摘要 & 章节

**价值**：长录屏（会议、课程）自动生成结构化摘要和章节标记。

**技术方案**：
```
字幕文本 → LLM (GPT/Claude) → 结构化摘要 + 章节时间点 → Studio 标记
```

**实施路径**：
1. 基于字幕文本（3.1 的产出）
2. 发送到 LLM API 进行分析
3. 返回：标题、摘要、关键章节（含时间戳）、TODO 列表
4. Studio 中展示章节导航，支持跳转

**输出示例**：
```json
{
  "title": "Q1 产品评审会议",
  "summary": "本次会议讨论了 Q1 目标完成情况和 Q2 规划...",
  "chapters": [
    { "time": "00:00", "title": "开场介绍" },
    { "time": "02:30", "title": "Q1 数据回顾" },
    { "time": "08:15", "title": "Q2 目标讨论" },
    { "time": "15:00", "title": "行动项确认" }
  ],
  "action_items": [
    "张三负责完成 A 功能开发",
    "李四准备 B 方案文档"
  ]
}
```

**商业模式**：
- 🆓 免费：查看摘要标题
- 💰 Premium：完整摘要 + 章节导航 + 行动项提取

**投入**：1-2 周（依赖 3.1 字幕能力）
**预计转化率**：3-5%（面向会议/教育场景用户）

### 3.3 ⭐ AI 智能标题 & 文件命名

**价值**：录制完成后自动生成有意义的文件名，替代 `rec_1707523661`。

**技术方案**：
```
当前页面标题 + 录制模式 + 时间 → 模板/LLM → 智能文件名
```

**实施路径**：
1. 录制时记录当前页面标题（`document.title`）
2. 结合录制模式、时长等元数据
3. 使用模板或轻量 LLM 生成描述性文件名
4. 用户可在 Studio 中编辑

**示例**：
```
录制 Google Docs 页面 → "Google Docs - Q1 报告编辑 - 2026-02-10"
录制 Zoom 会议 → "Zoom Meeting - 15min - 2026-02-10"
录制 Bug 复现 → "Bug Reproduction - Login Page - 2026-02-10"
```

**商业模式**：免费功能（提升体验，降低卸载）
**投入**：2-3 天
**预计体验提升**：显著降低文件管理门槛

## 四、P1 — 中期功能（1-2 月）

### 4.1 ⭐ AI 智能剪辑

**价值**：自动检测无效片段（静默、静止画面）并建议剪切点。

**技术方案**：
```
视频帧分析 → 场景变化检测 → 静默检测 → 智能剪切建议 → 用户确认
```

**功能细节**：
1. **静默检测**：分析音频波形，标记超过 N 秒的静默区间
2. **静止画面检测**：比较相邻帧差异，标记无变化区间
3. **场景切换检测**：识别画面大幅变化的时间点（用作章节分割）
4. **一键精剪**：移除所有静默/静止片段，保留有效内容

- 🆓 免费：静默检测标记

**投入**：3-4 周
**预计转化率**：8-12%

### 4.3 ⭐ AI 录制辅助（实时提示）

**价值**：录制过程中提供实时指导，提升录制质量。

**功能**：
1. 检测麦克风音量过低/过高，提示调整
2. 检测画面模糊/卡顿，提示网络/性能问题
3. 录制超长时提示分段
4. 检测鼠标不活跃，提示是否暂停

**商业模式**：免费功能（提升录制成功率和质量）
**投入**：1-2 周

## 五、P2 — 长期能力（2-4 月）

### 5.1 AI 画质增强

- 低分辨率录制后 AI 超分辨率提升
- 使用 WebGPU + ONNX Runtime 实现本地推理
- 技术难度高，但差异化明显

### 5.2 AI 音频降噪

- 录制后一键去除背景噪音
- 使用 RNNoise (WASM) 本地处理
- 投入中等，会议录屏场景需求大

### 5.3 AI 自动高亮

- 自动识别鼠标点击、键盘操作等关键动作
- 添加缩放、高亮等视觉效果
- 教程类录屏的差异化功能

### 5.4 AI 视频转文档

- 将录屏内容转化为图文教程/操作手册
- 自动截取关键帧 + 配合字幕文本
- 高价值 B2B 场景

## 十、总结

| 策略维度 | 建议 | 紧迫度 |
|----------|------|--------|
| 快速变现 | AI 字幕 + 内容摘要作为 Pro 功能 | ⭐⭐⭐⭐⭐ |
| 留存提升 | AI 智能命名 + 录制辅助（免费） | ⭐⭐⭐⭐⭐ |
| 差异化 | 本地 AI 优先 + 隐私友好 | ⭐⭐⭐⭐ |
| 长期壁垒 | WebGPU 本地推理 + 深度视频理解 | ⭐⭐⭐ |

> 💡 **核心结论**：AI 字幕生成是投入产出比最高的第一个 AI 功能。它同时满足三个目标：① 提供立竿见影的用户价值 ② 天然适合 Freemium 分层 ③ 技术实现相对成熟。建议作为 AI 商业化的第一步，在 3-5 周内完成 MVP 并上线。
>
> **隐私差异化**：与 Loom 等云端方案相比，本项目的核心优势是 "本地优先"。AI 策略应延续这一路线 —— 提供本地 AI 作为免费基础能力，云端 AI 作为 Premium 增值，让用户自主选择隐私与精度的平衡。
