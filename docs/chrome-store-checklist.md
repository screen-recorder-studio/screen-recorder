# Chrome 应用商店发布清单

发布 Chrome 扩展程序是一个多步骤的过程，需要仔细规划和准备。本清单将指导您完成所有必要的步骤，以确保您的扩展程序成功发布。

## 一、核心信息

这些是您在 Chrome 开发者信息中心提交扩展程序时需要提供的基本信息。

- **[ ] 名称:**
  - 您的扩展程序的名称。
  - 长度最多 45 个字符。
  - 确保名称独特且具有描述性。

- **[ ] 简短说明:**
  - 对您的扩展程序的简要概述。
  - 长度最多 132 个字符。
  - 将显示在 Chrome 应用商店的顶部。

- **[ ] 详细说明:**
  - 对您的扩展程序、其功能和优势的详细描述。
  - 使用清晰简洁的语言。
  - 突出显示主要功能和用例。
  - 您可以使用富文本格式（例如标题、项目符号）来提高可读性。

## 二、视觉资产

高质量的视觉资产对于吸引用户和传达扩展程序的功能至关重要。

- **[ ] 图标 (128x128):**
  - 您的扩展程序的图标。
  - 尺寸：128x128 像素。
  - 格式：PNG。
  - 将显示在 Chrome 应用商店和扩展程序管理页面中。

- **[ ] 屏幕截图 (1280x800 或 640x400):**
  - 至少一张，最多五张。
  - 尺寸：1280x800 像素或 640x400 像素。
  - 格式：PNG 或 JPEG。
  - 展示您的扩展程序的用户界面和功能。

- **[ ] 宣传图块 (440x280):**
  - 尺寸：440x280 像素。
  - 格式：PNG 或 JPEG。
  - 用于在 Chrome 应用商店中推广您的扩展程序。

- **[ ] 小型宣传图块 (440x280):**
  - 尺寸：440x280 像素。
  - 格式：PNG 或 JPEG。
  - 用于在 Chrome 应用商店中以较小的格式推广您的扩展程序。

- **[ ] YouTube 视频网址 (可选):**
  - 您的扩展程序的宣传视频的 URL。
  - 可以是演示、教程或宣传视频。

## 三、技术和政策合规性

确保您的扩展程序符合 Chrome 应用商店的政策对于避免被拒绝至关重要。

- **[ ] `manifest.json` 审查:**
  - 仔细检查您的 `manifest.json` 文件是否准确且完整。
  - 确保所有权限都是必要的，并已在您的描述中说明理由。

- **[ ] 隐私政策:**
  - 如果您的扩展程序处理用户数据，则必须提供隐私政策。
  - 隐私政策应托管在公开可访问的 URL 上。
  - 清楚地说明您收集了哪些数据以及如何使用这些数据。

- **[ ] 权限理由:**
  - 为您的扩展程序请求的每个权限提供明确的理由。
  - 解释为什么每个权限对于扩展程序的功能都是必要的。

## 四、营销和推广

在发布之前和之后推广您的扩展程序可以帮助您吸引更多用户。

- **[ ] 网站/登陆页面:**
  - 为您的扩展程序创建一个专门的网站或登陆页面。
  - 提供有关您的扩展程序的详细信息、屏幕截图和下载链接。

- **[ ] 社交媒体公告:**
  - 在社交媒体上宣布您的扩展程序的发布。
  - 使用相关的标签并与您的目标受众互动。

## 五、支持

为您的用户提供支持渠道对于建立信任和获得宝贵的反馈至关重要。

- **[ ] 支持电子邮件/网站:**
  - 提供用户可以联系您寻求支持的电子邮件地址或网站。
  - 及时回复用户查询和反馈。
