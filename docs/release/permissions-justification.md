## 权限理由

为了确保 Screen Recorder Studio for SAAS 能够提供其核心功能，我们需要请求以下 Chrome 权限。我们承诺仅将这些权限用于所述目的，绝不侵犯您的隐私。

*   **`desktopCapture`**
    *   **理由：** 此权限是录制功能的核心。它允许用户选择要录制的屏幕、应用程序窗口或浏览器标签页。没有此权限，我们将无法提供任何屏幕录制功能。

*   **`downloads`**
    *   **理由：** 此权限允许扩展程序将用户录制的视频文件保存到用户的本地下载文件夹。这是将录制内容交付给用户的唯一方式。

*   **`storage`**
    *   **理由：** 此权限用于在您的本地浏览器中安全地存储您的设置和偏好，例如默认的录制质量或您喜欢的录制模式。这有助于改善您的用户体验，让您无需每次都重新配置。所有数据都存储在本地，我们无法访问。

*   **`unlimitedStorage`**
    *   **理由：** 录制高清视频或长时间录制会产生非常大的文件。此权限允许扩展程序在用户本地存储超过 Chrome 默认 5MB 限制的视频数据。这对于确保用户可以录制和保存大型视频文件至关重要。

*   **`sidePanel`**
    *   **理由：** 此权限用于在浏览器侧边栏中显示扩展程序的用户界面。这为用户提供了一个方便、不干扰浏览内容的界面来控制录制、查看录制历史和管理设置。

*   **`activeTab`**
    *   **理由：** 当用户点击扩展图标时，此权限允许我们在当前活动的标签页上临时运行。这是实现元素选择和页面区域录制功能所必需的，因为它允许我们与页面的 DOM 进行交互。此权限是临时的，仅在用户与扩展交互时激活。

*   **`scripting`**
    *   **理由：** 此权限允许我们将脚本注入到用户选择的页面中。这是实现“元素录制”和“选区录制”功能的关键。通过注入脚本，我们可以识别用户选择的元素或区域，并仅录制该部分内容。我们不会使用此权限来读取或修改您页面上的任何其他内容。

*   **`tabs`**
    *   **理由：** 此权限用于获取有关浏览器中打开的标签页的信息。在录制过程中，我们需要此权限来跟踪被录制标签页的状态（例如，是否已关闭或刷新），以确保录制的稳定性和完整性。

*   **`offscreen`**
    *   **理由：** 从 Chrome 116 开始，Manifest V3 扩展程序需要使用 offscreen-documents 来处理一些耗时的后台任务，例如音频处理和视频编码。此权限允许我们在一个不可见的文档中执行这些操作，从而避免中断扩展程序的服务工作线程 (service worker) 并提高性能。

*   **`host_permissions: ["<all_urls>"]`**
    *   **理由：** 此权限有两个核心目的，旨在为用户提供最大的灵活性和功能性：
        1.  **随时随地录制：** 为了让用户能够在任何网页上无缝启动录制（尤其是“元素录制”和“选区录制”功能），扩展需要权限在所有页面上运行。这确保了用户无论在浏览哪个网站时，都能即时捕捉所需内容，而无需为每个网站单独授权。
        2.  **可选的在线发布：** 当用户选择将他们录制的视频发布到自己的视频托管平台时，此权限是必需的。它允许扩展程序安全地与用户指定的平台建立连接并传输视频文件。此过程完全由用户发起和授权。
    *   **我们郑重承诺：** 我们绝不会利用此权限在后台收集、监控或传输用户的任何浏览活动或数据。所有与外部服务器的通信都严格限制在用户主动发起的视频发布操作中。
